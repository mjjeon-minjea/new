# 텔레그램 비서 에이전트 선제적 능동형 대화 (Proactive Interaction) 도입 계획서 (R8)

사용자가 먼저 명령이나 질문을 던기 전까지 AI가 대기만 하고 노는 기존 수동형 비서 구조에서 벗어나, 로컬 Ollama AI(gemma4)의 비용이 전혀 들지 않는(무료 API) 강점을 120% 극대화하여 **실시간 금융 지표 급변동이나 최신 핵심 이슈 포착 시 비서가 텔레그램으로 선제적으로 말을 걸고(Proactive Alert & Chat) 관련 분석을 제안하는 능동 자율 비서 시스템**의 구현 계획서입니다.

## 1. User Review Required

> [!IMPORTANT]
> **수동형 비서 ➡️ 자율 능동형 에이전트로의 진화**
> AI가 단순히 사용자의 질문을 기다리는 것이 아닙니다. 백그라운드 스케줄러가 기동될 때마다 AI가 데이터를 스스로 평가하여, 중요 변동 사항이 있다면 **"민재님! 현재 ~한 상황인데 관련 심층 분석 보고서를 작성해 드릴까요? 😊"** 와 같이 선제적 채팅을 걸고 선택 버튼을 제안합니다.

> [!TIP]
> **로컬 AI(gemma4) 100% 무한 가동**
> 로컬 Ollama API는 완전 무료이므로 24시간 내내 백그라운드 지표를 감시하고 인공지능 분석을 돌려도 API 과금 우려가 전혀 없습니다. 이를 기반으로 한 시간 간격의 지표 변동 감지 및 AI 자율 판단 모듈을 탑재합니다.

* **주요 변경 사항**:
  1. **실시간 금융 변동 임계치 감시 (`check_financial_anomaly`)**:

  2. **AI 기반 선제적 대화 생성 (`generate_proactive_chat`)**:

     * 예: *"민재님! 방금 비트코인이 2% 급락하며 김치 프리미엄이 출렁이고 있어요. 관련 긴급 시장 영향 위키 리포트를 작성할까요? 📊"*
  3. **인라인 대화 제어 버튼 결합**:
     * AI의 선제적 메시지 밑에 인라인 키보드 버튼을 동적으로 생성하여 부착합니다:
       * `[🟢 응, 지금 바로 분석해줘]` -> 클릭 시 즉시 ChiefAgent의 릴레이를 돌려 분석 위키 초안을 합성하고 결과를 텔레그램 브리핑으로 송신.
       * `[🔴 아니야, 괜찮아]` -> 메시지를 확인 완료 상태로 변경.
  4. **백그라운드 감시 루틴 (`proactive_monitoring_loop`)**:
     * 12시간 주기 자동 브리핑 외에, 매 1시간마다 지표 및 최신 뉴스를 가볍게 훑으며 AI가 선제적으로 개입해야 하는지 판단하는 자율 감시 루틴을 가동합니다.

```python
async def scheduled_update_loop():
    """12시간 주기 자동 수집/분석 백그라운드 태스크"""
    global is_updating
    await asyncio.sleep(12 * 3600)
    
    while True:
        if not is_updating:
            is_updating = True
            try:
                agent = ChiefAgent()
                await asyncio.to_thread(agent.run_relay)
                await send_briefing_notification()
            except Exception as e:
                print(f"[!] [스케줄러 에러] 백그라운드 자동 실행 실패: {e}")
            finally:
                is_updating = False
        await asyncio.sleep(12 * 3600)
```

#### [After] `telegram_bot.py` (선제 감시 루프 및 AI 자율 선제 톡 발송 결합)
```python
last_checked_financial = {}

def check_financial_anomaly(current_data: dict) -> tuple:
    """이전 캐시 대비 유의미한 변동(비트코인 1.5% 이상, 환율 10원 이상 변동)이 있는지 감시합니다."""
    global last_checked_financial
    if not last_checked_financial:
        last_checked_financial = current_data
        return False, ""

    try:
        old_btc = last_checked_financial.get("btc_krw", 0)
        new_btc = current_data.get("btc_krw", 0)
        
        if old_btc > 0:
            btc_change = ((new_btc - old_btc) / old_btc) * 100
            if abs(btc_change) >= 1.5:
                direction = "급등" if btc_change > 0 else "급락"
                last_checked_financial = current_data
                return True, f"🪙 비트코인이 직전 대비 {abs(btc_change):.1f}% {direction} 중"

        old_usd = last_checked_financial.get("exchange_rate", 0)
        new_usd = current_data.get("exchange_rate", 0)
        if old_usd > 0 and abs(new_usd - old_usd) >= 10.0:
            direction = "급등" if new_usd > old_usd else "급락"
            last_checked_financial = current_data
            return True, f"💵 원/달러 환율이 {abs(new_usd - old_usd):.1f}원 {direction} 중"
    except Exception as e:
        print(f"[!] 변동성 평가 장애: {e}")
        
    return False, ""

async def send_proactive_alert(reason: str):
    """지표 이상이 감지되었을 때 AI가 먼저 대화를 구상하여 사용자에게 텔레그램 선제 톡을 송신합니다."""
    # 금융 지표 데이터 캐시 로딩
    fin_context = ""
    if FINANCIAL_DATA_PATH.exists():
        try:
            with open(FINANCIAL_DATA_PATH, "r", encoding="utf-8") as f:
                data = json.load(f)
                fin_context = f"현재 시황: 비트코인 {data.get('btc_krw', 0):,.0f}원, 환율 {data.get('exchange_rate', 0):,.2f}원"
        except Exception:
            pass

    prompt = (
        f"이상 징후 발생 원인: '{reason}'\n"
        f"상세 컨텍스트: {fin_context}\n\n"
        "당신은 개인 금융/지식 어시스턴트 'Antigravity 비서'입니다.\n"
        "이 상황을 인지하여 사용자 '민재'님에게 먼저 텔레그램 톡을 걸려고 합니다.\n"
        "상황의 긴박함이나 특이점을 다정하고 친근한 비서 톤의 한국어 구어체(이모지 포함)로 한 문장으로 요약하여 설명하고,\n"
        "관련 뉴스를 실시간으로 긴급 수집해서 종합 위키 분석 보고서를 만들어드릴지 정중히 제안하는 메시지를 작성하십시오.\n"
        "주의: 추가 설명 없이 오직 사용자에게 보낼 톡 메시지 내용 본문만 한 장으로 정확히 출력하십시오."
    )
    
    # AI가 보낼 메시지 구상
    ai_msg = await asyncio.to_thread(call_ollama, prompt, "당신은 능동적이고 선제적인 비서 비서 에이전트입니다.")
    
    if not ai_msg:
        ai_msg = f"🔔 *[긴급 변동 경보]*\n현재 `{reason}` 상황입니다. 실시간 관련 뉴스를 모아 긴급 종합 분석 위키를 가동해 드릴까요? 😊"

    # 인라인 조작 버튼 부착
    keyboard = [
        [
            InlineKeyboardButton("🟢 응, 지금 바로 분석해줘", callback_data="proactive_update_run"),
            InlineKeyboardButton("🔴 아니야, 괜찮아", callback_data="proactive_update_cancel")
        ]
    ]
    reply_markup = InlineKeyboardMarkup(keyboard)
    
    try:
        app = Application.builder().token(BOT_TOKEN).build()
        msg = await app.bot.send_message(
            chat_id=CHAT_ID,
            text=ai_msg,
            reply_markup=reply_markup,
            parse_mode="Markdown"
        )
        await log_chat_message(sender="bot", message_text=msg.text or "", session_id=str(CHAT_ID))
    except Exception as e:
        print(f"[!] 선제적 톡 발송 실패: {e}")

async def proactive_monitoring_loop():
    """매 1시간마다 로컬 금융 지표를 은밀하게 훑으며 AI 선제적 개입 여부를 평가하는 능동형 감시 루프"""
    print("[+] 24시간 실시간 AI 선제적 감시 루프 기동 완료.")
    while True:
        # 1시간 대기
        await asyncio.sleep(3600)
        
        # 0선 금융 스크래핑을 백그라운드에서 가볍게 1회 실행하여 실시간 지표 갱신
        try:
            from agents.chief_agent import ChiefAgent
            agent = ChiefAgent()
            # 0선 수집 파트만 가볍게 가동 (전체 릴레이가 아닌 단순 실시간 API 캐시 갱신)
            await asyncio.to_thread(agent.db_manager.collect_and_save_financial_data)
            
            if FINANCIAL_DATA_PATH.exists():
                with open(FINANCIAL_DATA_PATH, "r", encoding="utf-8") as f:
                    current_data = json.load(f)
                
                # 변동 평가
                has_anomaly, reason = check_financial_anomaly(current_data)
                if has_anomaly and reason:
                    # AI 선제적 알림 및 개입 시작!
                    await send_proactive_alert(reason)
        except Exception as e:
            print(f"[!] 백그라운드 선제 감시 주기 실행 오류: {e}")
```

## 4. Known Unknowns (자기완성도 선언 배제)

> [!WARNING]
> **Ollama 생성 시간 및 성능 한계**
> 1시간 주기의 백그라운드 평가 시 로컬 Ollama 모델의 연산 속도가 지연되면 봇의 Polling 루프 자체에 과부하를 주지 않도록 철저히 `asyncio.to_thread`를 사용하여 격리합니다.
> 
> **선제 톡의 빈도 한계**
> 시장의 가격 급변 시 1시간마다 계속해서 메시지가 송출되어 피로감을 유발하지 않도록, 한 번 경보가 발생하면 직전 경보 이후 최소 6시간 동안은 추가 경보를 억제하는 '피로 방지 쿨다운 필터(Cooldown Filter)'를 추가 설계합니다.

## 2. `config.py` 내 Obsidian Vault 공통 경로 정의

Obsidian 위키 마스터 및 원천 리포트 경로를 총괄하고 있는 `agents/shared/config.py` 내의 `RAW_DIR` 정의 분석입니다.

* **📂 `agents/shared/config.py` (L23)**
```python
23: RAW_DIR = VAULT_PATH / "raw"
```
* **상세 맥락**:
  - Line 21~22: `vault_path_str` 설정값(기본값 `"obsidian-vault"`)을 바탕으로 `VAULT_PATH`를 생성합니다.
  - Line 23: `VAULT_PATH` 아래의 `"raw"` 디렉토리를 `RAW_DIR`로 최종 바인딩합니다.
  - Line 35: `RAW_DIR.mkdir(parents=True, exist_ok=True)`를 통해 해당 디렉토리의 실재를 보장합니다.

## 3. 동시 저장 기능 구현을 위한 설계 방향 및 유의 사항

1. **임포트 경로 확보**: 
   - 4개 에이전트 모듈 상단에 `from agents.shared.config import RAW_DIR`을 추가하여 공통 경로 변수를 바인딩합니다.
2. **타겟 서브 디렉토리 개설**:
   - 전략 문서 전용 아카이빙 경로 `STRATEGY_DIR = RAW_DIR / "strategy"` 폴더가 존재하지 않을 수 있으므로, 파일 쓰기 전에 `STRATEGY_DIR.mkdir(parents=True, exist_ok=True)` 구문을 5선 기동 템플릿에 안전하게 수록합니다.
3. **이중 쓰기 수행**:
   - `session_dir` 하위에 1차 쓰기를 진행한 후, `STRATEGY_DIR` 하위에 동일한 파일명으로 2차 동시 쓰기(`write_text`)를 실행합니다.
   - 이때 Obsidian의 크로스링크 가독성을 극대화하기 위해 `STRATEGY_DIR / f"{session_dir.name}_macro_signal.md"` 처럼 타임스탬프를 파일명에 바인딩하여 덮어쓰기 충돌을 방지하거나, 사용자의 위키 추적 목적에 맞춰 고유한 시각화 파일명 포맷으로 동시 영속화하도록 유도합니다.

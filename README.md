## 🤖 3. 에이전트 명령어 및 기동 가이드

### 1) 봇 데몬 기동 및 단독 가동 명령어
* **비동기 텔레그램 봇 데몬 실행 (상시 Polling 모드)**:
  ```powershell
  python telegram_bot.py
  ```
* **단독 정기 요약 브리핑 푸시 (1회성 실행)**:
  ```powershell
  python telegram_bot.py --send-briefing
  ```

### 2) 7대 엔지니어링 표준 슬래시 명령어
사용자가 에이전트 패널에 아래 명령어를 치면 홈디렉토리 전역에 탑재된 스킬 라이브러리가 비동기로 탐색되어 기동됩니다.

| 명령어 | 단계 (Phase) | 역할 및 에이전트 행동 지침 |
| :--- | :--- | :--- |
| **`/spec`** | Define (명세 정의) | 요구사항을 엄격하게 질의응답하여 모호함을 없앱니다. |
| **`/plan`** | Plan (계획 수립) | 코드 작성을 절대 먼저 시작하지 않고, 작업 목록 및 검증 가능한 마일스톤 계획을 도출합니다. |
| **`/build`** | Build (점진적 구현) | 한 번에 전체를 짜지 않고, 작은 단위로 나누어 점진적으로 코드를 구현하고 결합합니다. |
| **`/test`** | Verify (테스트 검증) | TDD 원칙을 지키며, 예상 실패 테스트를 먼저 정의하고 이를 패스시키는 검증을 반복합니다. |
| **`/review`** | Review (코드 리뷰) | 시니어의 눈높이에서 아키텍처, 성능, 가독성, 예외 처리 등의 보안 및 품질 게이트를 검토합니다. |
| **`/code-simplify`** | Simplify (코드 간소화) | 에이전트 특유의 오버엔지니어링과 장황함을 쳐내고, 코드 가독성과 간결함을 극대화합니다. |
| **`/ship`** | Ship (배포 및 마감) | 변경점 로그 작성, 문서화, Git 버전 커밋 메시지 규칙 준수 하에 최종 배포를 마무리합니다. |

# 실시간 인터넷 뉴스 RAG 브리핑 기능 완수 보고서 (R2)

사용자가 최신 뉴스나 기업 이슈를 요청할 때, 봇이 관련 자료를 제출해 달라고 구걸하는 방식 대신 **실시간으로 구글 뉴스 RSS 피드를 스스로 검색하고 파싱하여 로컬 Ollama AI로 구조적 요약본을 만들어 송신하는 실시간 RAG 비서 기능**의 구현 및 배포를 완료하였습니다.

---

## 1. 변경된 핵심 아키텍처 및 소스 요약

### 1) 구글 뉴스 RSS 실시간 파서 (`fetch_realtime_news`)
* **기능**: 사용자가 입력한 키워드를 안전하게 URL 인코딩하여 구글 뉴스 RSS 스캔 주소에 연동합니다.
* **구조**: `requests` 및 `BeautifulSoup`을 통해 XML 노드를 정밀하게 분석하여 기사 제목, 링크, 배포처(언론사), 배포일, 요약문(`description`)을 스냅샷으로 추출합니다.

### 2) 지능형 의도 판별 및 키워드 추출 (`analyze_news_request`)
* **기능**: 사용자의 입력 자연어 문장이 일상 대화인지 뉴스/이슈 요약 요청인지 AI 모델(`gemma4:e4b`)을 기반으로 가볍게 단답형 JSON으로 판정하고, 최적의 검색용 명사 키워드를 도출합니다.
* **구조**: 단시간 내 연산이 완료되도록 템플릿 프롬프트를 구성하였으며, 오프라인 시에는 간단한 로컬 키워드 매칭 규칙으로 안전하게 폴백합니다.

### 3) 대화 핸들러 통합 (`text_message_handler`)
* **구조**: 뉴스 요청 판정이 나면 즉시 `🔍 [실시간 웹 검색 RAG 가동]` 알림 카드를 사용자에게 피드백하여 분석 중임을 명확히 시각화합니다.
* **출처 정보 크레딧 보존**: 생성된 AI 비서 요약 보고서 끝부분에 수집된 실제 뉴스 기사의 링크 및 언론사 크레딧을 함께 표기하여 투명성과 신뢰성을 확보했습니다.

---

## 2. 검증 결과 및 자가 진단 이력

### 1) 구문 검증 완료 (자가 진단 통과)
* 봇 소스코드 정적 검토를 위해 아래 구문 컴파일 테스트를 실행하였으며, 단 하나의 문법 에러나 예외 발생 없이 완벽히 통과했습니다:
  ```bash
  python -m py_compile telegram_bot.py
  ```

### 2) 봇 데몬 스위칭 및 재구동
* 기존 백그라운드 봇 데몬인 `task-1037`을 안전하게 종료(Kill) 처리했습니다.
* 최신 실시간 웹 뉴스 RAG 기능이 반영된 신규 봇 데몬을 백그라운드 프로세스(`task-1093`)로 정상 가동 및 대기 모드로 개시 완료했습니다.

---

## 3. Known Unknowns (자기완성도 선언 배제)

> [!WARNING]
> **외부 네트워크 신뢰성**
> 인터넷 RSS 리스닝 방식이므로, 로컬 환경의 일시적인 통신 단절이나 지연 현상이 있을 시 뉴스 목록이 정상적으로 반환되지 않을 우려가 존재합니다. 이 경우에도 프로세스가 다운되지 않도록 예외 차단 블록을 구현하였으며, 대기실 폴백 카드가 안정적으로 발송됩니다.
> 
> **AI 요약 가독성 유지**
> 입력받는 뉴스 콘텍스트의 내용이 무분별하게 길어지면 gemma4 모델의 컨텍스트 한계로 요약 퀄리티가 다소 낮아질 수 있어, 최신 뉴스 5개 이내로 뉴스 개수를 엄격히 조율해 퀄리티의 편차를 줄였습니다.

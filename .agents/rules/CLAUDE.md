# 안티그래비티 — Claude Code 협업 규칙

> 이 파일은 Claude Code(사용자와의 직접 대화)에만 적용됩니다.
> 자율 에이전트 파이프라인 규격은 `_company/_shared/_system.md`를 참조하세요.

---

## 보고서 저장 규칙

Claude가 문서를 작성하거나 검토하면, 아래 규칙에 따라 **자동으로 파일을 저장**합니다. 사용자가 별도로 요청하지 않아도 저장합니다.

### 저장 위치 및 파일명

| 문서 종류 | 저장 경로 | 파일명 형식 |
|---|---|---|
| Claude 작성 모든 보고서 | `claude_reprot/` | `YYYY-MM-DD_[제목]_검토_R{n}.md` |

- `{n}`은 0부터 시작하는 수정 번호입니다. 같은 제목의 파일이 이미 있으면 R{n+1}로 저장합니다.
- 날짜는 오늘 날짜(YYYY-MM-DD)를 사용합니다.
- 제목의 공백은 `_`로 대체합니다.
- `antigravity_report/`는 안티그래비티 전용 디렉토리입니다. **Claude는 절대 저장하지 않습니다.**

### 검토 보고서 헤더 형식

```markdown
# [제목] 검토

**작성일**: YYYY-MM-DD
**검토 대상**: `[검토한 파일명]`
**검토 방법**: [검토 방법 설명]
**수정번호**: R{n}
```

## 프로젝트 구조 요약

```
_company/          → 자율 에이전트 파이프라인 (건드리지 않음)
agents/            → 에이전트 Python 코드
antigravity_report/ → 안티그래비티 전용 (Claude 접근 금지)
claude_reprot/      → Claude 작성 보고서 전용 (오타 있는 폴더명 — 그대로 유지)
obsidian-vault/    → Obsidian 위키 데이터
```

---

## 행동 원칙

- `_company/` 내부 파일은 사용자가 명시적으로 요청할 때만 수정합니다.
- 계획서나 문서를 검토할 때는 실제 코드/파일을 직접 대조한 뒤 작성합니다.
- 보고서 작성 후 저장 경로를 사용자에게 알립니다.

## 문제점 상세

### 오류 1 — "에이전트 폴더 5개 파일" 주장 (사실 오류)

보고서는 에이전트 폴더마다 `config.md`, `prompt.md`, `memory.md`, `tools.md`, `skills/` **5개**라고 명시했다.

**실제 구조:**
```
_agents/business/
  config.md
  prompt.md
  memory.md
  tools.md
  goal.md          ← 보고서에서 누락
  rag_mode.txt     ← 보고서에서 누락
  skills/
  tools/           ← 일부 에이전트에 별도 tools/ 폴더도 존재
```

`goal.md`와 `rag_mode.txt`를 아예 인식하지 못했다. "줄 단위로 분석"했다는 주장과 모순된다.

---

### 오류 2 — `approval_gate`를 완성된 기능으로 소개 (심각한 오류)

보고서는 텔레그램 승인 버튼 워크플로를 핵심 강점으로 대대적으로 소개했다.

**실제 `_agents/ceo/tools.md` 내용:**
```markdown
### `approval_gate` _(예정)_
- 아직 구현되지 않은 도구입니다. 로드맵에 있으며 향후 버전에서 추가 예정.
```

**미구현 로드맵 항목**을 현재 동작하는 기능처럼 묘사했다. "강력히 도입 제안"한 핵심 기능이 실제로는 존재하지 않는다. 벤치마킹 보고서의 치명적 오류다.

---

### 오류 3 — 에이전트 수 불일치 미검증

`_system.md` 헤더는 **"7명의 에이전트"** 라고 쓰여 있지만, 같은 파일 안의 목록과 실제 폴더는 **10명**이다.

보고서는 10명이라고 맞게 썼지만 원본 시스템 내부의 이 불일치를 인지하지 못하고 그냥 지나쳤다. 원본 시스템 자체의 문서 결함을 발견할 기회를 놓쳤다.

---

### 오류 4 — v4.0 제안에서 자율도 레벨 임의 설정

**보고서가 제안한 v4.0 구조:**
```
chief_agent/tools.md  # 자율도 수준 (Default: 3 - Auto)  ← 근거 없음
db_manager/tools.md   # 자율도 수준 (Default: 3 - Auto)  ← 근거 없음
```

**실제 Connect-AI 기본값:**
CEO를 포함한 모든 에이전트의 `AUTONOMY_LEVEL: 2`. `tools.md`에도 `2 (Draft) ⭐ 권장 기본값`이라고 명시되어 있다.

벤치마킹을 한다면서 원본 철학과 반대되는 값을 제안했다.

---

### 오류 5 — `10_Wiki/` 폴더 미검증

메모리 위계에 `10_Wiki/ (외부 지식 데이터베이스)`를 5번째 계층으로 소개했지만, 실제 `.connect-ai-brain` 구조 어디에도 해당 폴더는 존재하지 않는다. `_system.md`에 언급만 있고 실제 구현은 없는 상태다.

---

### 오류 6 — v4.0 폴더 구조가 원본 네이밍 무시

**원본 Connect-AI:**
```
_agents/
_shared/
```

**보고서 v4.0 제안:**
```
agents/
  shared/
```

언더스코어 prefix를 제거했다. Connect-AI 구조에서 언더스코어는 일반 콘텐츠 폴더와 시스템 디렉토리를 구분하는 설계 의도가 담겨 있다. 또한 제안 구조에는 에이전트별 `memory.md`, `goal.md`, `rag_mode.txt`가 아예 빠져 있다.

---

### 오류 7 — 과도한 홍보 어조 (객관성 부재)

기술 검토 보고서임에도 아래 표현들이 다수 사용됐다:

- "무결점 v3.0", "버그가 완치된 상태"
- "완벽한 사상과 디테일"
- "차원이 다른", "혁신적인"
- "강력히 제안드립니다"

분석 보고서가 아니라 광고 문구에 가깝다. 원본 시스템의 미구현 항목, 내부 불일치 등 단점은 전혀 언급하지 않았다.

---

## 핵심 결론

Gemini가 실제로 파일을 깊이 읽기보다 폴더 구조와 `_system.md` 개요만 보고 나머지를 추정해서 작성한 것으로 판단된다. 특히 `approval_gate`가 미구현 상태라는 것을 놓친 것은 실사 부재의 증거다.

**도입 검토 전 선행 확인이 필요한 항목:**
1. `approval_gate` 구현 상태 (현재 미구현 — 텔레그램 승인 워크플로 직접 구현 필요)
2. `rag_mode.txt`의 역할 및 우리 시스템 적용 방안
3. `goal.md` (에이전트 개별 목표 파일) 활용 전략
4. `10_Wiki/` 폴더 부재 — 지식베이스 연동 방식 별도 설계 필요
